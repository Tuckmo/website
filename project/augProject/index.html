AugProject

 

AugProject was a project that I contributed to during my capstone unit COMP356 for my Bachelor of Information Technology at Macquarie University.

The main goal of this project was to bring more awareness to the art located around Macquarie University by creating a augmented reality GPS tracking mobile application similar to Pokemon Go that allowed the users to visit art pieces and learn more about them inside a somewhat competitive game.

The basic functionality of this project was to create a educational scavenger like hunt across Macquarie University using the browser on mobile phones using GPS to locate mapped locations, tracking which ones had been visited or not. Upon finding these locations map markers would be generated on the Google Map to interact with, with the ability to pull up more information about the artist and object you had found, answer a question on the object that had been found or view the 3d model for that object.

Unfortunately the app will only work with the GPS coordinates that are currently in the Google Sheet which are all located around Macquarie University. The best way to get an idea of the app is to either look at the documentation below or the presentation that we made at the end of the project.

App Overview and Features

AugProject

Documentation & Presentation Outline

The project is an interactive augmented reality treasure hunt application, which works completely on the web using only HTML, CSS and Javascript for cross-browser support.

Main Goals of the project:

  • Geolocation and finding of placed locations within a certain radius to the user
  • Accessibility for all devices - restrained us to web technologies rather than native app support.
  • Notifying a user when they are near one of our locations, and to be able read past locations they have found without going back to the location that they found
  • Database administration -
  • Admins without programming skill can manage the location data using more common skills like spreadsheets
  • User engagement to the application
  • Questions
  • Scores from finding locations

Overall Features

  • Dynamically locating the user and notifications when a target location
  • A dynamic updating user map using google maps
  • 3D models displayed at each location with interactive elements
  • Interactive user questions
  • User tracking - ability to re-view found locations
  • Responsive design
  • Database administration using google maps
  • A nodejs backend to circumvent browser security features, implement api function calls and serve static pages.
